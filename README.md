### マルチモジュール化したい

- `app`モジュールの追加
   appモジュールは、アプリケーションのエントリーポイントとして機能し、他のモジュールを統合します。
   ・アプリケーションレベルの設定を含む
   ・依存関係の注入の設定
   ・メインアクティビティとナビゲーション構造の定義
- `feature`モジュールを作成し配下に画面を配置
  - `feature:taskList`
  - `feature:taskInput`
  機能ごとに独立したモジュールを作成することで、コードの分離と再利用性が向上します。
- `build-logic`モジュールを追加しコンベンションプラグインをマルチモジュール環境で適用
   ビルド設定の一元管理とカスタマイズを可能にします。
- `core`モジュールを作成し配下に以下を追加
  - `core:database`
  - `core:model`
  共通のコンポーネントやユーティリティを含むコアモジュールは、アプリケーション全体で再利用可能なコードを提供します。

### テスト実装について
- 各モジュールにtestディレクトリを追加し、ユニットテストを実装
- androidTestディレクトリで統合テストとUIテストを実装
